Nesse caso digamos que eu tenho uma pagina aberta qque faz parte do meu sistema,
mas ue quero ue mesmo que o usuario não esteja logado, ele possa ter acesso ao conteudo 
dela.Isso é um cenario normal em sistema de vitrine onde o comprador pode ver os produtos,
mas não pode alterar os preços deles, ou seja conteudo condicionado.

No caso do meu projeto eu criei uma pagina de contatos e coloquei o link dela na minha home page.

Home page (protegida)
página de contatos(aberta) 

Esse trecho de codigo mostraa a minha view que tem 2 tipos de conteudo um texto que será mostrado,
somente para usuarios autenticados e outros texto qque so aparece se o usuario tiver feito login.
o sistema sabe que o usuario esta autenticado quando o @auth foi preechido com o login dele então 
ele testa se não estiver vazio ele vai mostrar o texto.

Quando o usuario não faz o login mas vcissita a pagina quam recebe os dados dele e a diretiva '@guest' 
como o nome ja diz se trata de uma visitante que noss não sabemos quem é então com base na minha regra,
ele só terá acesso ao texto para usuarios não autenticados. 

ATENÇÃO!
  Não confundir com autorização, a logica se trata somente de identificar o usuário ou não identificar ele.

<div class="container mt-5">
    <div class="row">
        <div class="col text-center">
            <span class="display-3">PÁGINA DE CONTATOS</span>
            <hr>
            <p>Esta é uma página de contatos que não requer autenticação.</p>
            <p>Você pode adicionar informações de contato aqui.</p>
        </div>

        @auth
           <p class="display-6 text-success">
             Esse texto só é exibido para usuários autenticados.
           </p>
        @endauth
        @guest
           <p class="display-6 text-danger">
             Esse texto só é exibido para usuários não autenticados.
           </p>
        @endguest
    </div>
</div>
